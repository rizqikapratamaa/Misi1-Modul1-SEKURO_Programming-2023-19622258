# 3. GitHub: Branch
~~~
Branch adalah cabang yang digunakan untuk membuat snapshot
tanpa mengganggu jalur utamanya.
Branch berguna untuk fitur experimental, 2 orang menggunakan
repository yang sama.

Merging adalah proses menyatukan branch menjadi satu kesatuan utuh. Hal ini biasanya dilakukan untuk menggabungkan kode yang dikerjakan oleh beberapa orang yang berbeda.

Checkout adalah berpindah ke branch/commit lainnya

Pull request adalah meminta pemilik repo untuk "mengambil"
perubahan yang telah dilakukan.
~~~
